# OpenUpdater 프로젝트 이력 관리 (HISTORY)

이 파일은 프로젝트의 주요 결정 사항, 소스 수정, 빌드 및 배포 이력을 기록하여 프로젝트의 방향성을 유지하기 위해 사용됩니다.

---

## 📅 2026-04-23

### 🔄 [v1.0.0.7] 셀프 업데이트(Self-Update) 기능 도입
- **작업 내용**: 앱 실행 시 깃허브 최신 릴리즈를 체크하여 자신을 자동으로 업데이트하는 기능 추가.
- **메커니즘**: 임시 배치 파일(Batch)을 이용한 프로세스 교체 방식 적용으로 단일 파일 무결성 유지.
- **개선**: `GitHubDownloadService`에 JSON 파싱 로직을 추가하여 릴리즈 정보를 정확히 추출함.

### 🚀 [v1.0.0.6] 빌드 안정화 및 최종 배포
- **작업 내용**: WPF 빌드 안정성을 위해 `PublishTrimmed` 설정을 영구 제거하고 버전을 업데이트함.
- **최적화**: 단일 파일 압축(`EnableCompressionInSingleFile`) 및 ReadyToRun 최적화는 유지.

### 🚀 [v1.0.0.5] 실행 파일명 변경 및 최종 브랜딩
- **작업 내용**: 실행 파일명을 `WpfAutoUpdater.exe`에서 프로젝트 명칭인 `OpenUpdater.exe`로 변경.
- **빌드 수정**: WPF에서 지원하지 않는 `PublishTrimmed` 설정을 비활성화하여 빌드 오류 해결.

### ✨ [v1.0.0.4] 사용성 개선 및 앱 최적화 (App Diet)
- **용량 최적화**: `.NET Trimming` 및 `Compression` 설정을 적용하여 실행 파일 용량을 획기적으로 줄임.
- **설정 기능 강화**: 업데이트 완료 후 실행할 파일명을 사용자가 `UpdatePath.ini`에서 직접 설정할 수 있도록 기능 추가.
- **UI/UX 개편**: 
    - `업데이트 시작` 버튼 강조 (녹색).
    - `취소` 버튼을 `종료`로 변경.
    - 자동 실행 대상 파일명을 UI에 명시하여 사용자 혼란 방지.

### 🤖 [v1.0.0.3] 빌드 검증 시스템 도입
- **검증 자동화**: 빌드된 `.exe` 파일의 속성(FileVersion)이 릴리즈 태그와 일치하는지 확인하는 PowerShell 스크립트 단계를 GitHub Actions에 추가.
- **안전장치**: 버전 불일치 시 빌드를 강제 실패 처리하여 잘못된 배포 방지.

### 🔧 [v1.0.0.2] 빌드 설정 수정 및 단일 파일 배포
- **단일 파일 배포**: 모든 DLL 의존성을 포함하는 `SingleFile` 설정 적용.
- **자립형 빌드**: .NET 런타임이 없는 환경에서도 실행 가능하도록 `SelfContained` 옵션 적용.
- **아키텍처 명시**: `win-x64` 전용 빌드 파이프라인 구축.

### 📦 [v1.0.0.1] WPF 마이그레이션 및 초기 릴리즈
- **WinForms to WPF**: 기존 WinForms 기반 업데이트 프로그램을 WPF로 마이그레이션.
- **기능**: FTP, GitHub Release, Direct URL 소스 지원 기능 이식.
